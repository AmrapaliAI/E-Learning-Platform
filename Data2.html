Summary

Database Management System

Database Examples with Explanations

Stored Procedure


CREATE PROCEDURE GetStudentDetails
    @StudentID INT
AS
BEGIN
    SELECT Name, Course, Marks
    FROM Students
    WHERE StudentID = @StudentID;
END;

A stored procedure is a reusable block of SQL code saved in the database. In this example, the procedure GetStudentDetails takes a student ID as input and returns the student’s name, course, and marks. This avoids rewriting the same query multiple times and improves efficiency.

Exception Handling


BEGIN TRY
    INSERT INTO Students (StudentID, Name, Course)
    VALUES (101, 'Rahul', 'Computer Science');
END TRY
BEGIN CATCH
    PRINT 'Error occurred: ' + ERROR_MESSAGE();
END CATCH;

Exception handling ensures that errors are managed gracefully. The TRY block attempts to insert a new student record. If an error occurs (for example, inserting a duplicate StudentID), the CATCH block executes and prints the error message. This prevents the system from crashing and provides useful feedback.

SQL Query


SELECT Name, Course
FROM Students
WHERE Marks > 75
ORDER BY Name ASC;

A query is used to retrieve data from the database. This example selects the Name and Course of students who scored more than 75 marks. The results are sorted alphabetically by name. Queries like this are commonly used to filter and organize data for reporting.
