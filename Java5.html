Module 6: JDBC & Database Connectivity (Detailed)

1. Introduction to JDBC

  • JDBC (Java Database Connectivity) is an API for connecting Java applications to relational databases.
  • Provides a standard interface for executing SQL queries and managing results.
  • Part of the java.sql package.

2. Steps in JDBC

  1. Load the database driver.
  2. Establish a connection using DriverManager.
  3. Create a Statement or PreparedStatement.
  4. Execute SQL queries.
  5. Process results using ResultSet.
  6. Close the connection.

3. Key Classes & Interfaces

  • DriverManager: Manages database drivers and connections.
  • Connection: Represents a session with the database.
  • Statement: Executes static SQL queries.
  • PreparedStatement: Executes parameterized queries, prevents SQL injection.
  • ResultSet: Stores results of queries.

4. CRUD Operations

  • Insert: Add new records.
  • Select: Retrieve records.
  • Update: Modify existing records.
  • Delete: Remove records.
Connection con = DriverManager.getConnection(
    "jdbc:mysql://localhost:3306/testdb", "root", "password");
Statement stmt = con.createStatement();
ResultSet rs = stmt.executeQuery("SELECT * FROM students");
while(rs.next()) {
    System.out.println(rs.getString("name"));
}

5. Transactions

  • Group multiple operations into a single unit of work.
  • Use commit() to save changes.
  • Use rollback() to undo changes if errors occur.

6. Best Practices

  • Always close connections, statements, and result sets.
  • Use PreparedStatement to prevent SQL injection.
  • Implement connection pooling for efficiency.
  • Handle exceptions properly with try-catch-finally blocks.

Key Takeaways

  • JDBC enables Java applications to interact with databases seamlessly.
  • CRUD operations form the backbone of database programming.
  • Transactions ensure data integrity and consistency.
  • Best practices improve performance and security.
