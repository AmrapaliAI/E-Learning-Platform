Course Syllabus: AWS Fundamentals

Module: AWS Compute Services

This module explores the core compute services in AWS, which provide the processing power to run applications, workloads, and containerized environments in the cloud.

1. Amazon EC2 (Elastic Compute Cloud)

  • Definition: Virtual servers in the cloud with customizable CPU, memory, and storage.
  • Key Features: Auto Scaling, Elastic Load Balancing, multiple instance types.
  • Use Cases: Web hosting, enterprise applications, batch processing.

2. AWS Lambda

  • Definition: Serverless compute service that runs code in response to events.
  • Key Features: No server management, automatic scaling, pay-per-execution pricing.
  • Use Cases: Event-driven apps, API backends, real-time file processing.

3. Amazon Lightsail

  • Definition: Simplified compute service for small workloads.
  • Key Features: Pre-configured instances, predictable pricing, easy setup.
  • Use Cases: Blogs, websites, small business apps.

4. Amazon ECS (Elastic Container Service)

  • Definition: Container orchestration service for Docker containers.
  • Key Features: Integration with EC2 and Fargate, scalable clusters.
  • Use Cases: Microservices, containerized workloads.

5. Amazon EKS (Elastic Kubernetes Service)

  • Definition: Managed Kubernetes service.
  • Key Features: Fully compatible with upstream Kubernetes, integrates with IAM and VPC.
  • Use Cases: Kubernetes-based applications, hybrid deployments.

6. AWS Fargate

  • Definition: Serverless compute engine for containers.
  • Key Features: No need to manage servers, scales automatically.
  • Use Cases: Running containers without managing EC2 instances.

7. Best Practices

  • Choose the right instance type for workload efficiency.
  • Use Auto Scaling to handle variable traffic.
  • Leverage IAM roles for secure access to compute resources.
  • Monitor performance with CloudWatch metrics.

8. Hands-On Activities

  1. Launch an EC2 instance and connect via SSH.
  2. Deploy a Lambda function triggered by an S3 event.
  3. Set up a Lightsail instance for a WordPress blog.
  4. Run a containerized app using ECS or EKS.
  5. Experiment with Fargate to deploy containers without managing servers.
