Summary

DBMS is software that manages databases, enabling efficient storage, retrieval, and manipulation of data. It supports data integrity, security, and concurrent access, forming the backbone of modern applications.

SQL Language Concepts Explained

SQL, or Structured Query Language, is the standard language used to interact with relational databases. It allows users to create, read, update, and delete data efficiently. SQL is widely supported across database systems such as MySQL, Oracle, PostgreSQL, and SQL Server.

A SQL query is a request for data from the database. Queries can filter, sort, and combine information using commands like SELECT, WHERE, ORDER BY, and JOIN. For example, a query can retrieve all students who scored above 75 marks and list them alphabetically.

SQL also supports Data Definition Language (DDL) commands such as CREATE, ALTER, and DROP. These commands define and modify the structure of database objects like tables, views, and indexes. DDL ensures that the database schema is properly designed and maintained.

Data Manipulation Language (DML) commands include INSERT, UPDATE, DELETE, and SELECT. These commands allow users to modify and retrieve data stored in tables. DML is the most commonly used part of SQL in everyday applications.

SQL provides transaction control through commands like COMMIT, ROLLBACK, and SAVEPOINT. Transactions ensure that multiple operations are executed as a single unit of work, following the ACID properties: Atomicity, Consistency, Isolation, and Durability.

Exception handling in SQL, especially in systems like SQL Server, is managed using TRY...CATCH blocks. This allows developers to catch errors during query execution and handle them gracefully without crashing the application.

SQL also supports stored procedures and functions, which are reusable blocks of code stored in the database. These improve performance and maintainability by encapsulating complex logic that can be executed with a single call.

SQL Language Categories with Examples

DDL (Data Definition Language)

DDL commands define and modify the structure of database objects such as tables, views, and indexes. Common commands include CREATE, ALTER, and DROP. 


CREATE TABLE Students (
    StudentID INT PRIMARY KEY,
    Name VARCHAR(50),
    Course VARCHAR(50),
    Marks INT
);

ALTER TABLE Students ADD Email VARCHAR(100);

DROP TABLE Students;

DML (Data Manipulation Language)

DML commands are used to manipulate data stored in tables. Common commands include INSERT, UPDATE, and DELETE. 


INSERT INTO Students (StudentID, Name, Course, Marks)
VALUES (101, 'Rahul', 'Computer Science', 85);

UPDATE Students
SET Marks = 90
WHERE StudentID = 101;

DELETE FROM Students
WHERE StudentID = 101;

DQL (Data Query Language)

DQL is used to query and retrieve data from the database. The main command is SELECT, often combined with WHERE, ORDER BY, and JOIN. 


SELECT Name, Course, Marks
FROM Students
WHERE Marks > 75
ORDER BY Name ASC;

TCL (Transaction Control Language)

TCL commands manage transactions in a database. They ensure data integrity using COMMIT, ROLLBACK, and SAVEPOINT. 


BEGIN TRANSACTION;

UPDATE Students
SET Marks = Marks + 5
WHERE Course = 'Computer Science';

COMMIT;

-- If something goes wrong, use:
ROLLBACK;

DCL (Data Control Language)

DCL commands control access to data in the database. Common commands include GRANT and REVOKE. 


GRANT SELECT, INSERT ON Students TO User1;

REVOKE INSERT ON Students FROM User1;
