Azure Fundamentals - Core Overview

What is Microsoft Azure?

Microsoft Azure is a cloud computing platform and infrastructure provided by Microsoft. It enables organizations to build, deploy, and manage applications through a global network of data centers.

Why Use Azure?

  • Scalability: Easily scale resources up or down based on demand.
  • Cost Effective: Pay only for what you use, no upfront hardware cost.
  • High Availability: Applications run across multiple Availability Zones.
  • Security: Built-in security services and compliance standards.
  • Global Reach: Data centers available worldwide.

Azure Global Infrastructure

  • Regions: Separate geographic areas like East US, West Europe, Southeast Asia.
  • Availability Zones: Multiple isolated data centers inside a region.
  • Edge Locations: Used for faster content delivery via Azure Front Door and CDN.

Azure Shared Responsibility Model

Azure follows a shared responsibility model where security responsibilities are divided:

  • Azure Responsibility: Security of the cloud (hardware, data centers, networking).
  • Customer Responsibility: Security in the cloud (data, OS, applications, IAM).

Core Azure Service Categories

  • Compute: Virtual Machines, Functions, App Services, AKS
  • Storage: Blob Storage, Disk Storage, File Storage, Archive
  • Networking: VNet, DNS, Front Door, VPN Gateway
  • Databases: SQL Database, Cosmos DB, Synapse, Redis Cache
  • Security: Azure AD, Key Vault, Security Center, Sentinel
  • Monitoring: Monitor, Application Insights, Log Analytics

Key Azure Concepts

  • Elasticity: Automatic scaling of resources.
  • Fault Tolerance: System continues working even after failures.
  • High Availability: Minimal downtime using multi-AZ design.
  • Pay-As-You-Go: No long-term contracts.

Common Azure Use Cases

  • Web and application hosting
  • Backup and disaster recovery
  • Big data processing and analytics
  • Machine learning and AI applications
  • Mobile and game development
