Course Syllabus: AWS Fundamentals

Module: AWS Databases

This module explores AWS database services for structured and unstructured data, analytics, and caching.

1. Amazon RDS (Relational Database Service)

  • Definition: Managed relational database service supporting MySQL, PostgreSQL, Oracle, SQL Server, and MariaDB.
  • Key Features: Automated backups, scaling, multi‑AZ deployment, read replicas.
  • Use Cases: Enterprise applications, transactional systems, e‑commerce platforms.

2. Amazon DynamoDB

  • Definition: Fully managed NoSQL database service.
  • Key Features: Single‑digit millisecond latency, auto‑scaling, global tables.
  • Use Cases: Gaming, IoT, real‑time applications.

3. Amazon Redshift

  • Definition: Data warehousing and analytics service.
  • Key Features: Columnar storage, parallel query execution, integration with BI tools.
  • Use Cases: Business intelligence, big data analytics, reporting dashboards.

4. Amazon ElastiCache

  • Definition: In‑memory caching service supporting Redis and Memcached.
  • Key Features: Low latency, high throughput, managed clusters.
  • Use Cases: Session management, caching queries, speeding up web apps.

Module: AWS Security

This module covers AWS security services that protect resources, manage identities, and ensure compliance.

1. AWS IAM (Identity and Access Management)

  • Definition: Service for managing user identities and permissions.
  • Key Features: Policies, roles, MFA, fine‑grained access control.
  • Use Cases: Secure access to AWS resources, least privilege enforcement.

2. AWS Shield

  • Definition: Managed DDoS protection service.
  • Key Features: Standard and Advanced tiers, automatic detection and mitigation.
  • Use Cases: Protect web applications from denial‑of‑service attacks.

3. AWS KMS (Key Management Service)

  • Definition: Service for creating and controlling encryption keys.
  • Key Features: Key rotation, integration with AWS services, compliance support.
  • Use Cases: Encrypt sensitive data, secure storage, compliance requirements.

4. AWS CloudTrail

  • Definition: Logging and auditing service for AWS API calls.
  • Key Features: Event history, monitoring, compliance auditing.
  • Use Cases: Security auditing, troubleshooting, governance.
