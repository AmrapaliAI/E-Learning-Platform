Introduction to Java

Module 4: Collections Framework (Detailed)

1. Introduction

  • Collections provide built-in data structures to store and manipulate groups of objects.
  • They replace the need for manual arrays and linked lists with reusable implementations.
  • Part of java.util package.

2. Core Interfaces

  • List: Ordered collection, allows duplicates (e.g., ArrayList, LinkedList).
  • Set: Unordered collection, no duplicates (e.g., HashSet, TreeSet).
  • Map: Key-value pairs (e.g., HashMap, TreeMap).
  • Queue: FIFO structure (e.g., PriorityQueue, LinkedList).

3. Implementations

  • ArrayList: Dynamic array, fast random access.
  • LinkedList: Doubly linked list, efficient insert/delete.
  • HashSet: Stores unique elements, backed by hash table.
  • TreeSet: Sorted set, backed by a tree structure.
  • HashMap: Key-value pairs, fast lookup.
  • TreeMap: Sorted map, maintains order of keys.

4. Generics

  • Enable type safety in collections.
  • Example: ArrayList<String> list = new ArrayList<>();

5. Iterators

  • Used to traverse collections.
  • Example:
ArrayList<String> names = new ArrayList<>();
names.add("Amrapali");
names.add("Java");
for(String n : names) {
    System.out.println(n);
}

6. Utility Classes

  • Collections: Provides methods like sort(), reverse(), shuffle().
  • Arrays: Provides methods to manipulate arrays (e.g., Arrays.sort()).

Key Takeaways

  • Collections simplify data handling in Java.
  • Different interfaces and implementations suit different use cases.
  • Generics and Iterators make collections
