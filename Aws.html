AWS Fundamentals - Core Overview

What is Amazon Web Services (AWS)?

Amazon Web Services (AWS) is a secure and scalable cloud computing platform provided by Amazon. It offers on-demand access to computing power, storage, networking, databases, and many other services over the internet with a pay-as-you-go pricing model.

Why Use AWS?

  • Scalability: Easily scale resources up or down based on demand.
  • Cost Effective: Pay only for what you use, no upfront hardware cost.
  • High Availability: Applications run across multiple Availability Zones.
  • Security: Built-in security services and compliance standards.
  • Global Reach: Data centers available worldwide.

AWS Global Infrastructure

  • Regions: Separate geographic areas like Mumbai, Virginia, London.
  • Availability Zones: Multiple isolated data centers inside a region.
  • Edge Locations: Used for faster content delivery via CloudFront.

AWS Shared Responsibility Model

AWS follows a shared responsibility model where security responsibilities are divided:

  • AWS Responsibility: Security of the cloud (hardware, data centers, networking).
  • Customer Responsibility: Security in the cloud (data, OS, applications, IAM).

Core AWS Service Categories

  • Compute: EC2, Lambda, ECS, EKS
  • Storage: S3, EBS, EFS, Glacier
  • Networking: VPC, Route 53, CloudFront
  • Databases: RDS, DynamoDB, Redshift
  • Security: IAM, KMS, Shield, CloudTrail
  • Monitoring: CloudWatch, AWS Config, X-Ray

Key AWS Concepts

  • Elasticity: Automatic scaling of resources.
  • Fault Tolerance: System continues working even after failures.
  • High Availability: Minimal downtime using multi-AZ design.
  • Pay-As-You-Go: No long-term contracts.

Common AWS Use Cases

  • Website and application hosting
  • Backup and disaster recovery
  • Big data processing and analytics
  • Machine learning and AI applications
  • Mobile and game development
